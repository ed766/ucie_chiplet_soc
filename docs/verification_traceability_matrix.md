# Verification Traceability Matrix

This matrix is the quickest review path for the chiplet DV story. It connects
major requirements to stimulus, checkers, assertions, coverage, and generated
evidence. The detailed pass/fail source of truth remains the CSV reports under
`chiplet_extension/reports/`.

## Reading Order

1. Start with `docs/project_metrics.md` for current counts.
2. Use this matrix to find the evidence for a feature.
3. Open `docs/bug_diary.md` for debug stories and injected-fault examples.
4. Open `docs/true_cross_coverage_summary.md` for non-gating interaction coverage.
5. Open `docs/axi_lite_coverage_summary.md` for optional AXI-Lite CSR wrapper coverage.
6. Use `docs/uvm_status.md` for the optional UVM methodology and RAL lane.
7. Use `docs/firmware_soc_verification.md` for CPU-driven subsystem verification.
