# Machine Timer, WFI, and Counter Verification

The firmware integration top contains an always-on 64-bit machine timer at `0x1A0-0x1AC`. The core implements `mie.MTIE`, read-only `mip.MTIP/MEIP`, machine-timer cause `0x80000007`, external-before-timer priority, `WFI`, and writable `mcycle/minstret` low/high CSRs.

Measured evidence:

- `10 / 10` compiled-C scenarios, including a focused high-half CSR and architectural EBREAK edge matrix.
- `16 / 16` trace-derived timer/WFI/counter points.
- `22 / 22` same-window crosses.
- `10 / 10` traces pass independent cycle/retirement-delta checking of counter reads, writes, rollover, and write precedence.

The scenarios cover future and already-expired compares, masked-pending-then-enable, a timer becoming pending during an APB wait, simultaneous external/timer priority, timer and DMA wake sources, WFI across modeled SLEEP, counter write/rollover precedence, and firmware-measured DMA/IRQ latency. The trap handler distinguishes interrupt sources and moves `mtimecmp` forward (or disables it) before `MRET`.

Evidence is generated by:

```bash
make -C chiplet_extension timer-wfi-check
```

Canonical reports are `timer_wfi_summary.csv`, `timer_wfi_coverage_summary.csv`, `timer_wfi_cross_coverage_summary.csv`, and `timer_counter_semantics.csv` under `chiplet_extension/reports/`.
